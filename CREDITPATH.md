# CREDITPATH.md

## Product Name
CreditPath

## Purpose
CreditPath is FundReady's native credit improvement engine.

It turns the existing FundReady diagnosis into a personalized, trackable, step-by-step roadmap that helps users strengthen their credit profile, improve lender readiness, and unlock better capital outcomes over time.

CreditPath is not a generic credit repair feature.
It is not a referral handoff.
It is not a separate business.
It is part of FundReady's borrower-readiness operating system.

## Why CreditPath Exists
FundReady already knows a large amount about a user's credit and lender-readiness profile through the assessment and existing engine outputs.

The platform already collects or derives signals related to:
- personal credit score range
- utilization
- derogatories
- bankruptcy
- collections and charge-offs
- tax liens
- inquiries
- business credit depth

That means FundReady already starts with something most credit products do not have:
a pre-diagnosed user with a scored, ranked problem set.

CreditPath exists to turn that diagnosis into:
- clarity
- action
- momentum
- measurable progress
- stronger capital readiness

## Core Product Thesis
CreditPath helps users move from:
- confusion to understanding
- shame to agency
- guesswork to roadmap
- weak profile to stronger lender readiness
- denial risk to bankable progression

It should always feel like:
- a guided improvement system
- a lender-readiness companion
- a progress engine
- a trust-centered financial product

It must never feel like:
- a scammy credit repair funnel
- a vague self-help tool
- a generic AI chatbot
- a disconnected side product
- a shame-based compliance workflow

## Core User Promise
CreditPath helps you understand what is suppressing your credit profile, shows you the highest-leverage actions to take next, and tracks your progress toward becoming more bankable.

## Who CreditPath Serves
### 1. First-Time Strategic Applicant
A founder or owner who has mostly bootstrapped, has little funding experience, and wants to get it right the first time.

### 2. Denied but Viable Owner
An owner who has been denied, underfunded, or offered poor terms and needs a clearer recovery path.

### 3. Growth-Oriented Operator
A business owner who can access some capital but wants to graduate to stronger and cheaper products.

### 4. Overlooked or Underserved Founder
A founder with real potential but less financial literacy, less banking access, or thinner supporting profile signals.

### 5. Advisor / Channel Partner
A tax professional, broker, consultant, or advisor who wants a structured way to guide client improvement.

## Core Product Outcomes
CreditPath should help users:
- understand where they stand
- know what is hurting lender confidence
- know what to do next
- feel capable of acting
- stay engaged long enough to see measurable progress
- connect profile improvement to real capital outcomes
- build confidence over time

## Behavioral Design Rules
CreditPath must follow these rules:

### Rule 1: Show only the top 3 actions
Never overwhelm the user with a long problem list.
Always rank and show the top 3 actions only.

### Rule 2: Make the first action tiny
The first action should ideally take under 10 minutes.

### Rule 3: Show the mechanism, not just the promise
Do not say "this will fix your score."
Explain why a specific action matters and how it affects the profile.

### Rule 4: Tie action to financial upside
Whenever possible, connect progress to lender readiness or capital opportunity using safe estimate language.

### Rule 5: Build identity, not shame
Do not frame the user as broken.
Frame progress as becoming more bankable, more lender-ready, and more financially credible.

### Rule 6: Guidance before judgment
CreditPath must feel like a skilled guide, not a compliance interrogation.

## CreditPath Product Layers
### Layer 1 -- Diagnosis
Already exists in FundReady.

CreditPath must reuse existing FundReady signals and outputs wherever possible.
It must not create a second intake or a duplicate scoring system.

### Layer 2 -- Personalized Repair Roadmap
CreditPath generates a ranked action plan for the specific user.

The ranking must prioritize:
1. likely impact on readiness
2. time to result
3. ease of execution
4. relevance to the user's actual blocker set

### Layer 3 -- Progress Tracking and Milestones
Every completed action should strengthen the user's sense of progress through:
- progress states
- milestone badges
- visible movement
- next milestone framing
- updated action ranking

## Required CreditPath Sections
### 1. Credit Health Overview
Shows the user's current profile issues in clear priority order.

### 2. Your Repair Roadmap
Shows the top 3 actions ranked by impact and time-to-result.

### 3. Progress Tracker
Shows what was completed, what changed, and what is unlocked next.

### 4. Credit Tools
Provides structured tools that help users take action, not just read advice.

## Required Credit Tools
CreditPath v1 should support these tools conceptually:

- Utilization payoff calculator
- Dispute letter workflow
- Business tradeline starter list
- Inquiry aging tracker

## Credit Data Source Strategy
CreditPath must support a 3-layer data-input model.

### Layer A -- Self-Reported Assessment Data
This is the default and required starting point for every user.

CreditPath must work from:
- unified assessment data
- current deterministic engine outputs
- current readiness and credit-related outputs

This layer can power:
- Credit Health Overview
- top-3 roadmap
- milestone model
- broad estimate ranges
- basic coaching

### Layer B -- Uploaded Credit Report
Users may optionally upload a bureau report or monitoring report to improve precision.

This layer should improve:
- issue-level specificity
- dispute-letter specificity
- simulator quality
- monthly analysis precision

### Layer C -- Optional Monitoring / API Integration
A monitoring or bureau-linked partner integration may be added later.

This is optional and should not block v1.

This layer should improve:
- ongoing refresh
- recurring progress reporting
- automated delta tracking
- premium workflow value

## Product Rule for Data Strategy
CreditPath v1 must work without requiring a monitoring partner.

The build order is:
1. self-reported assessment first
2. uploaded report second
3. optional monitoring/API integration third

## User-Facing Confidence Disclosure
CreditPath must always know which data mode is driving its outputs.

The product must distinguish between:
- self-reported data
- uploaded report data
- connected monitoring data

And user-facing copy must reflect that difference clearly.

## What CreditPath Must Reuse
CreditPath must reuse and stay aligned with:
- the existing assessment
- existing engine outputs
- Dashboard patterns
- Status Reports patterns
- FORGE patterns
- Compliance / progress patterns
- existing trust and estimate guardrails

## What CreditPath Must Not Introduce
CreditPath must not introduce:
- a second scoring system
- duplicate status logic
- disconnected upgrade states
- AI-first business logic
- hype-heavy "repair your bad credit fast" messaging
- certainty or guarantee language
- a separate user journey that ignores the current FundReady progression model

## Success Criteria
CreditPath is successful when:
- users can understand their situation quickly
- users see exactly 3 next actions
- users take the first action faster
- users feel less overwhelmed
- users see progress over time
- CreditPath strengthens retention and upgrade logic without harming trust
